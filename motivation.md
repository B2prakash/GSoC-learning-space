## Who I am
I'm a final-year Master's student in Computer Science/Engineering. My primary language is Python, and I've worked across the ML stack — data pipelines, model training, and experiment tooling. I'm comfortable with NumPy, pandas, and visualization libraries like Matplotlib and Plotly. Most of my projects have been research-adjacent, which means I've spent a lot of time making complex outputs interpretable, both for myself and for others.

## Why Mesa
I was looking for a Python open-source project where I could make meaningful contributions, not just fix typos. Mesa stood out because it sits at an interesting intersection: it's a scientific tool, but it has real software engineering challenges — API design, performance, and especially visualization. I hadn't used agent-based modeling much before, but once I started reading the codebase and running examples, I realized how much depth there is. The Mesa 3.x rewrite also means the project is actively evolving, which makes it a good time to get involved.

## What I want to learn
I want to understand how agent-based models are actually structured — how agents, spaces, schedulers, and data collection fit together in practice, not just in the docs. I also want to learn how to work on a library used by researchers, where backward compatibility and API clarity matter a lot. On the visualization side, I want to understand how Mesa's Solara-based visualization works end-to-end, and where the current approach falls short for users.

## Where I want to go
I want to contribute to Mesa's visualization layer — making it easier to build interactive, informative model visualizations without a lot of boilerplate. I think there's room to improve how users can customize what they see during a simulation run, and I'd like to work on that. Longer term, I want to be a regular contributor, not just a GSoC participant — someone who reviews PRs, helps triage issues, and understands the codebase well enough to have opinions on design decisions.
